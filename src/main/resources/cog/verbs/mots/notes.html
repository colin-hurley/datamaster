Notes on MotS Verbs

AI Alignment

 The alignment values used by the AI alignment verbs is the same alignment that is specified in an actor's AI definition file. In MotS, the following values are used for alignment:

Alignment Usage
-1.0 AI with this alignment are hostile toward the player (E.g. Imperials)
0.0 AI with this alignment are neutral and do not attack (E.g. Civilians)
1.0 AI with this alignment are allied with the player (E.g. Rebels)

Bubbles

MotS introduces the ability to create bubbles. A bubble is a spherical region around a thing that moves with the thing and sends cog messages when the player enters (enterbubble) or exits (exitbubble) the bubble region.

Bubbles are always created with jkCreateBubble(), but a thing's bubble can be destroyed by either jkDestroyBubble() or some other event that removes the thing from the game. A bubble can be resized after creation with jkSetBubbleRadius().

Every bubble has a type associated with it. A bubble's type is initially set at creation time by jkCreateBubble(), but the type can later be changed for existing bubbles via jkSetBubbleType(). Although the verbs will accept negative values for the bubble type, not all parts of the mechanism handle the negative values properly, so a non-negative integer should always be used for the bubble type to avoid problematic behavior.

The jkGetFirstBubble() and jkGetNextBubble() verbs can be used to locate all bubbles that currently exist in the level.
